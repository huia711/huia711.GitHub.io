ADVERSARIAL EXAMPLES IN THE PHYSICAL WORLD

2023-12-06 / 5 min read

Title

ADVERSARIAL EXAMPLES IN THE PHYSICAL WORLD
#adversarial_example

Venue

ICLR 2017

Problem

Is it still possible to craft adversarial examples and perform adversarial attacks in the physical world?

Motivation

  • All previous work has assumed the adversary can feed data directly into the machine learning classifier.
  • But in the physical world, many practical machine learning systems perceiving data through various sensors, rather than digital representation. Is it still possible to craft adversarial examples and perform adversarial attacks on them?

Significance/Contribution

This finding demonstrates the possibility of adversarial examples for machine learning systems in the physical world.

Method

1. METHODS OF GENERATING ADVERSARIAL IMAGES

  • Trying to increase the cost of the correct class

  • Sufficient for number of classes is small and all classes are highly distinct from each other

    1. Fast method
      • Fast: not require an iterative procedure
      • Xadv=X+ϵsign⁡(∇XJ(X,ytrue ))\boldsymbol{X}^{a d v}=\boldsymbol{X}+\epsilon \operatorname{sign}\left(\nabla_X J\left(\boldsymbol{X}, y_{\text {true }}\right)\right)Xadv=X+ϵsign(∇X​J(X,ytrue ​))
    2. Basic iterative method
      • require an iterative procedure
      • X0adv=X,XN+1adv=ClipX,ϵ{XNadv+αsign⁡(∇XJ(XNadv,ytrue))}\boldsymbol{X}_0^{a d v}=\boldsymbol{X}, \quad \boldsymbol{X}_{N+1}^{a d v}=C l i p_{X, \epsilon}\left\{\boldsymbol{X}_N^{a d v}+\alpha \operatorname{sign}\left(\nabla_X J\left(\boldsymbol{X}_N^{a d v}, y_{t r u e}\right)\right)\right\}X0adv​=X,XN+1adv​=ClipX,ϵ​{XNadv​+αsign(∇X​J(XNadv​,ytrue​))}

  • But ImageNet has a much larger number of classes and the varying degrees of significance in the difference between classes.

    1. Iterative least-likely class method
      • make an adversarial image which will be classified as a specific desired target class
      • yLL=arg⁡min⁡y{p(y∣X)}y_{L L}=\underset{y}{\arg \min }\{p(y \mid \boldsymbol{X})\}yLL​=yargmin​{p(y∣X)}
2. EXPERIMENT
  • Destruction rate
    • the influence of arbitrary transformations on adversarial images
    • d=∑k=1nC(Xk,ytrue k)C(Xadvk,ytrue k)‾C(T(Xadvk),ytrue k)∑k=1nC(Xk,ytrue k)C(Xadvk,ytrue k)‾d=\frac{\sum_{k=1}^n C\left(\boldsymbol{X}^k, y_{\text {true }}^k\right) \overline{C\left(\boldsymbol{X}_{a d v}^k, y_{\text {true }}^k\right)} C\left(T\left(\boldsymbol{X}_{a d v}^k\right), y_{\text {true }}^k\right)}{\sum_{k=1}^n C\left(\boldsymbol{X}^k, y_{\text {true }}^k\right) \overline{C\left(\boldsymbol{X}_{a d v}^k, y_{\text {true }}^k\right)}}d=∑k=1n​C(Xk,ytrue k​)C(Xadvk​,ytrue k​)​∑k=1n​C(Xk,ytrue k​)C(Xadvk​,ytrue k​)​C(T(Xadvk​),ytrue k​)​
  • The experimental procedure
    1. Print the Image
    2. Take a photo of the printed image using a cell phone camera
      • introduces nuisance variability
    3. Automatically crop and warp validation examples from the photo
      • use QR codes in the corners of the photo
      • they would become squares of the same size as source images
  • Two sets of experiments
    1. Average case
      • randomly select
      • estimate how often an adversary would succeed on randomly chosen photos
    2. Prefiltered case
      • select images that all clean images are classified correctly, and all adversarial images are classified incorrectly
      • how often an adversary would succeed when the adversary can choose the original image to attack
